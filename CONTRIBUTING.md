# Contributing to Hydrix

Thank you for your interest in contributing to **Hydrix**!
We welcome contributions that help improve quality, performance, stability, and documentation.

This document outlines the guidelines for contributing to this project.

You can contribute in several ways:

- Reporting bugs
- Suggesting enhancements
- Improving documentation
- Submitting code changes (Pull Requests)

## üêû Reporting Bugs

When reporting a bug, please include:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Environment details (OS, .NET version, database, etc.)
- Relevant logs or stack traces, if available

---

## üí° Suggesting Enhancements

Feature requests are welcome. Please provide:

- A clear description of the problem the feature solves
- A proposed solution or approach
- Any potential impact on performance or compatibility

---

## üîß Code Contributions

### Fork & Branch
- Fork the repository
- Create a branch from `main` or `develop`
- Use a clear and descriptive branch name  
  Example: `feature/support-in-parameters`

### Coding Guidelines
- Follow existing code style and patterns
- Keep methods small and readable
- Prefer explicit, clear abstractions over clever code
- Ensure cognitive complexity remains reasonable
- Write meaningful commit messages

### Tests
- Add or update tests when applicable
- Ensure all tests pass before submitting a PR
